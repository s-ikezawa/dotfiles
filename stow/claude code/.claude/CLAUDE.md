# CLAUDE.md

## Common guidelines

- 常に日本語で会話してください。
- ファイルに書き込む場合は日本語で記述してください。
- 文字コードは常にUTF-8を使用してください。

## Coding guidelines

- コメントは日本語で記述してください。
- 原則としてテスト駆動開発（TDD）で進めてください。
  - 期待される入出力に基づきテストコードを作成する。
    - 実際のコードは書かずにテストコードのみを作成してください。
  - テストを実行し、失敗することを確認する
    - この時点ではテストが失敗することが正しいです。
  - テストが失敗することを確認した後、テストをパスさせる実装を進めてください。
  - 実装中はテストコードは変更せずに実際のコードを修正し続けてください。
  - すべてのテストが通過するまで繰り返してください。
  - すべてのテストが通過したらビルドが必要な言語の場合はビルドを実行してください。
  - ビルドが問題なく完了したらコミットしてください。
- クラスやメソッドのコメントには仕様を記述してください。
  - 実装に使用する言語ごとにドキュメント出力用のコメントの書き方がある場合はそれに沿った書き方で記述してください。

## Commit message guidelines

- コミットメッセージは日本語で記述してください。

## Issue guidelines

- `GitHub Issue` は日本語で記述してください。

## Pull request guidelines

- `GitHub PullRequest` は日本語で記述してください。
