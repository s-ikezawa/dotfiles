## 主なワークフロー

### ソフトウェアエンジニアリングタスク

機能の実装、リファクタリング、コード説明などを求められた場合は以下を順に実施します。

1. **理解 (Understand) :** ユーザーの要望と関連コードベースの文脈を把握し、理解します。
2. **計画 (Plan) :** 課題解決のための整合的で根拠のある計画を立案し、極めて簡潔かつ明確な形でユーザーに共有します。計画の一部として単体テストを記述して自己検証ループを構築することを計画します。
3. **実装 (Implement) :** **重要な制約事項**に従って実装します。
4. **検証 (Test) :** 実装した単体テストを実行し実装に問題がないかを確認します。テストのコマンドは `README` や `package.json` などから特定し、仮定しないでください。
5. **検証 (Standards) :** プロジェクト固有のビルド・リント・型チェックコマンド（例: `tsc`、`npm run lint`、`ruff check .`）を実行し、品質を担保してください。コマンドが不明の場合はユーザーに確認してください。

## MCP Servers

MCPサーバーを利用するのが良い場合は積極的に利用してください。

### Context7

フレームワークやライブラリなどのドキュメントがまとまっているMCPサーバーです。  
フレームワークやライブラリを利用する際は利用しているバージョンを指定すると対象のドキュメント内容を取得することができます。  
実装やバグ修正などに積極的に利用してください。

## CLI Tools

### Gemini CLI

Web検索を必要とするときに `gemini` コマンドが利用できる場合は以下のコマンドで実行してください。

```bash
gemini -p "WebSearch: {Content}"
```
